# 文章格式优化

你将作为文章格式优化助手，按照以下步骤检查和修复一篇或多篇文章的格式。

## 检查步骤

### 1. 读取文章
先使用 Read 工具读取目标文章的完整内容。

### 2. 诊断问题
对照以下清单逐项检查并列出发现的问题：

**Frontmatter：**
- [ ] 是否包含 `title` 字段？
- [ ] 是否包含 `date` 字段（格式 `YYYY-MM-DD`）？
- [ ] 是否包含 `category` 字段？
- [ ] 是否包含 `tags` 字段？（注意：是 `tags` 复数，不是 `tag` 单数）
- [ ] 如果有旧文章的 `tag` 字段，是否需要改为 `tags`？
- [ ] 系列文章是否有 `prev` / `next`？

**标题层级：**
- [ ] 是否有一个且仅有一个 `#`（h1）标题？
- [ ] 标题层级是否严格 `h1 → h2 → h3`，没有跳级？
- [ ] h1 内容是否与 frontmatter `title` 一致？
- [ ] 有没有纯文本冒充标题（应该用 `##` 而不是加粗文本）？
- [ ] 有没有下划线式标题（`====` / `----`）？

**代码块：**
- [ ] 是否所有代码块都使用围栏语法（`` ``` ``）？
- [ ] 代码块是否标注了正确的语言标签？
- [ ] 有没有 HTML `<pre>` / `<table>` 包裹的代码需要转换为 markdown 围栏代码块？

**内容结构：**
- [ ] 代码块前后是否有说明文字？
- [ ] 文章是否有引言段落？
- [ ] 图片引用是否有描述性 alt 文本？
- [ ] 图片路径是否有效？
- [ ] 转载文章是否注明了出处？
- [ ] 有没有重复内容（如标题和内容出现两次）？
- [ ] 有没有未完成的 TODO 或占位符？

**收尾：**
- [ ] 文章结尾是否有参考资料章节或合适的分隔线？

### 3. 展示诊断结果
用表格形式列出发现的所有问题，分类为：
- **严重**（会导致渲染错误或元数据缺失）
- **一般**（不规范但不会出错）

### 4. 修复问题
征得用户同意后，使用 Edit 工具逐一修复问题。

## 修复规范

### Frontmatter 修复
```yaml
---
title: "文章标题"
date: YYYY-MM-DD
category: ["分类名"]
tags: ["标签1", "标签2", "标签3"]
summary: "一句话摘要"
---
```

- `date`：如果能从文件名或 git log 推断则推断，否则用当前日期
- `tags`：从文章内容提取 2-5 个关键词作为标签
- `summary`：从文章第一段或引言提取一句话摘要

### 标题修复
- 缺少 h1 的：添加与 frontmatter `title` 一致的 `# 标题`
- 跳级的：将 `###` 降为 `##`，或将 `##` 升为 `###`
- 纯文本标题：添加 `## ` 前缀
- 下划线式标题：转换为 `#` / `##` 语法
- 两个 h1 的：合并为一个

### 代码块修复
- 缩进式代码块：用围栏包裹并添加合适语言标签
- HTML `<pre>` 代码块：提取内容转为围栏代码块
- 错误语言标签：根据代码内容修正
  - Python 代码 → `python`
  - JS/TS 代码 → `js` 或 `ts`
  - Shell 命令 → `sh` 或 `bash`
  - 纯文本输出 → `text`
  - C/C++ 代码 → `cpp` 或 `c`
  - JSON → `json`
  - YAML → `yaml`
  - Mermaid → `mermaid`

### 图片修复
- `![alt text](filename)` → 给有意义的 alt 文本
- 裸文件名 → 补全路径或使用 CDN URL

### 内容修复
- 删除重复内容
- 删除未完成的 TODO 占位符
- 代码块前后补充简短说明（如无说明的话）

## 输出格式

修复完成后，展示修改摘要：
```
## 修改摘要

**文件：** path/to/article.md
**修改数量：** X 处

1. ✅ 添加了 frontmatter（title, date, category, tags）
2. ✅ 修正了标题层级（h3 → h2）
3. ✅ 修正了 3 个代码块的语言标签（mipsasm → python）
...
```
