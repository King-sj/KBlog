# KBlog 项目概述

KBlog 是基于 VuePress 2 + vuepress-theme-hope 架构搭建的个人技术博客。作者 KSJ，站点地址 https://bupt.online。

博客定位：个人知识库 + 技术分享，坚持 "Talking is cheap, show me your code."

项目使用 pnpm 作为包管理器，具体格式规范见 `.claude/rules/article-formatting.md`。

---

## 语言与写作风格

### 技术文章

**精准、结构化、教程式**。遵循"是什么 → 为什么 → 怎么做 → 总结"的脉络。技术术语保留英文或中英并列（如"Singleton 模式"），不强行翻译约定俗成的英文术语。

典型结构：
```
# 标题
定义/背景（一句话）
## 为什么需要 / 背景与目标
## 核心概念 / 工作原理
## 示例 / 代码
## 注意事项 / 常见问题排查
## 应用场景
## 总结
## 参考
```

### 项目复盘

**口语化、自嘲式**。频繁使用 `~~删除线~~` 包裹自嘲、吐槽或补充说明。坦诚记录踩坑和翻车经历。文末"结语"总结得失。

### 随笔

**极度简短、哲理化**。通常一两段，以日常小事为引子拔高到人生感悟，标题常为成语或谚语。

### 读书笔记

**精炼、萃取式**。只记录核心观点，不追求完整复述。用 `::: tip` 等提示框补充个人见解。

---

## 惯用修辞与口头禅

| 表达 | 使用场景 |
|------|---------|
| `~~内容~~` | 自嘲、吐槽、不重要补充 |
| `emmm...` | 迟疑、无奈或思考中 |
| `qWq` | 卖萌/无奈（类似 QAQ） |
| `众所周知` / `事实上` | 引出论点、加强语气 |
| `TODO` | 标记未完成的文章 |
| `::: tip` / `::: important` | 强调要点 |

---

## 受众与语调

- 面向自己（个人知识库，很多文章本质是学习笔记和备忘）
- 面向有基础技术背景的全栈开发同好
- 不追流量，不写水货
